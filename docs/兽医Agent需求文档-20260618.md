## 0. 一句话
做一个面向**宠物主人**的兽医 AI 助手:能像兽医一样**追问到方向清楚**、**自动调用我们已有的宠物数据**(少问基础题)、**记得这位主人和这只宠物**、给出**有据、分级、安全**的辅助诊断与处置建议;**同时也能答非医疗的养宠问题(行为学、饲养技巧、日常护理)**;**还能看图——读化验报告、读病历、(最后)读片子**——医疗部分辅助参考、不取代线下兽医。

## 1. 背景与定位
- **产品上下文**:爪连智能宠物健康项圈 + 小程序。我们已有大量该宠物的真实数据(画像/生理指标/活动/定位/告警),以及正在建的**健康算法服务 v2.0**(输出健康风险域 + 证据)和**现有 AI 对话域**(`apps/ai`)。兽医 agent 应当**站在这些之上**,不是从零的聊天机器人。
- **第一阶段对象 = 宠物主人(消费端,小程序内)**。语气=大白话;责任=参考非诊断、强制就医兜底。
- **能力天花板 = 到处置/用药建议**(Mao 定)。⚠️ 这是最高风险档,**§4 临床安全架构是本文第一性要求**,不是附录。
- **本文是需求**:要它"做到什么 / 能用什么 / 不能越什么线 / 怎么算合格"。**框架、模型、怎么实现、怎么做安全工程,归同事**。

## 2. 核心能力需求(对应你的五条诉求,逐条可验)

### R1 辅助诊断级(分诊 → 方向 → 鉴别 → 处置/用药)
四层递进,每层都带安全约束(见 §4):
1. **分诊 + 紧急度**:判断"要不要看医生 / 多紧急(可在家观察 / 尽快就诊 / 急诊)"。
2. **方向提示**:可能的大方向(如"偏消化 / 偏关节 / 偏呼吸")。
3. **鉴别方向**:列可能病因 + 各自依据(不下定论)。
4. **处置 / 用药建议**:护理 + 用药**方向**。**🔒 已定(Mao 2026-06-18):不输出具体剂量数字——涉及用药一律提示"按药品使用说明书或遵从兽医指导"**(见 §4.3/§4.4)。
- 每一层输出都必须:**给出"为什么这么判"的依据**(来自数据 / 算法证据 / 用户回答)+ **始终附"建议线下兽医确认"**。

### R2 主动追问,直到大方向确认
- 像问诊一样**结构化追问**:先看已有数据(R3)→ 算出"还缺哪些关键信息"→ **只问缺的**,一次别问太多(每轮 1–3 个最相关问题)。
- **停下来的条件要明确**:大方向的置信度达到阈值 / 鉴别已收敛到少数 / 触发急症红线(立即转就医)/ 用户信息已问尽仍不足(诚实说"需要线下检查")。**不能无限追问、也不能没问清就下结论。**
- 反例(要避免):一上来问一堆已经知道的(品种、年龄、体重);或绕圈重复问。

### R3 调用既有数据,少问基础题
- **凡是我们已有的,绝不再问用户。** 进对话先拉该宠物的画像 + 最近健康指标 + 算法风险 + 告警 + 历史(见 §5),用进推理与追问。
- 规则:**"系统已知 → 直接用并可向用户复述确认;系统未知 → 才问。"** 例:不问"你家狗多重",而是"系统记录它约 12kg,最近一次是 X 时间,对吗?"
- 数据缺失/过期要会判断(如体重是半年前的 → 可确认一下),不能盲信也不能无视。

### R4 对客户/宠物的记忆
- **三层记忆**:
  - **宠物级**:历次咨询主诉、出现过的症状/发作、给过的建议、后续转归(吃了药好没好);慢性问题(如关节、心肺趋势)长期记。
  - **主人级**:沟通偏好、养宠情况(多只宠物分别记)、过往关注点、风险偏好(爱紧张 / 佛系)。
  - **会话级**:能接上上一次没说完的,不每次从零开始。
- **建在现有之上**:`apps/ai` 已有 `AIChatSession/AIDialogMessage`(对话持久化);**`aiuserprofile` 现状是"读了从不写"(AI 人设恒空,见迁移评审)——这次要把它真正写起来**,作为主人级记忆载体。
- 记忆要**可被用户查看 / 纠正 / 删除**(合规)。

### R5 准确的输出
- **有据**:每个结论能指到依据(哪条数据 / 哪个算法证据 / 用户哪句回答)——复用算法服务的 `evidence_contribution`(逐指标证据)做"可解释",不是黑箱下结论。
- **带不确定度**:该说"不确定 / 需要进一步检查"时就说,禁止为了像专家而编造("未标定模型的 0.9 ≠ 90% 真会发生")。
- **不幻觉、不越界**:超出**养宠范围(健康+行为+饲养)**的问题礼貌拒答 + 引导;不编造数据里没有的"检查结果"。
- **一致**:同样的情况给一致的判断(可回归测试)。

### R6 非医疗养宠咨询(行为学 + 饲养 + 日常护理)
除医疗外,还要能答**非医疗的养宠问题**——高频、低风险、高粘性的入口:
- **行为学**:乱叫/拆家/扑人/护食/不亲人/分离焦虑行为/社会化/如厕与行为训练等。
- **饲养技巧**:喂什么/喂多少/换粮/不同生命阶段(幼·成·老)营养、体重管理、零食、饮水。
- **日常护理**:运动量、洗护、口腔、环境布置、季节注意。

**要求**:
1. **同样吃既有数据 + 记忆做个性化**(承 R3/R4):喂量按"该宠物的物种/品种/年龄/体重/活动量"算,不是泛泛网文;行为建议结合算法的**行为/分离焦虑信号**(算法框架模板 B/C)+ 活动/定位数据。这正是我们比通用宠物 GPT 强的地方——**它懂这只具体的宠物**。
2. **🔴 跨域安全钩子(关键,新增)**:非医疗问题里**常藏着医疗红旗**——"它最近不爱吃饭 / 突然乱咬转圈 / 暴瘦 / 一直舔某处"听着像饲养/行为问题,实则可能是病。**每一条输入都要先过 §4.2 的急症/红旗筛查,命中就切到分诊/就医,绝不当普通饲养问题答过去。** 这是把非医疗能力接进医疗 agent 最容易出事、也最该守的点。
3. **饲养里的安全**:复用 §4.2 的**有毒食物黑名单**(巧克力/葡萄/木糖醇/洋葱等)主动拦截 + 警告;不给无依据的极端食谱(纯生骨肉/绝食减肥等)。
4. **准确/不编**:行为/饲养有较多业界共识(不像诊断那样无真值),但仍要**基于主流科学实践、不传谣**;拿不准就说("这块有争议,建议咨询专业行为训练师/营养方向兽医")。

### R7 视觉/多模态输入(读报告 → 读病历 → 读片子,按此优先级)
让用户拍照/上传,agent 看图理解。**按难度与风险分三档,也是落地优先级(Mao 定:片子最后):**
1. **报告(化验单/检查报告)— 先做、最易**:OCR 出文字+数值(血常规/生化/尿检等),对照参考范围标异常,作为强证据接进诊断推理(R1/R5)。
   - ⚠️ **数值必须可核对、不能盲信 OCR**:读出的关键数值/剂量要**回显给用户确认**("我读到 ALT=120,对吗?")。OCR 错一个数可能直接误导判断甚至用药——这些数还会喂进剂量逻辑。
2. **病历(既往病历/处方/诊断记录)— 次做、协同价值最高**:OCR + 结构化抽取(疫苗史、既往诊断、慢性病、用过的药)→ **直接喂进记忆(R4)与"既有数据"(R3)**。强协同:**用户上传一次病历,agent 永久记住这只宠物的病史,以后更不用反复问**。
3. **片子(X 光 / B 超等影像)— 最后做、最难、最高风险**:影像判读。
   - 🔒 **已定(Mao 2026-06-18):第一阶段不放开影像判读,等之后学习/有验证再做。** 第一阶段最多接收图片,但只"建议把片子交给专科兽医",**不出任何影像判读结论**。理由:消费端 + 无临床数据 + 影像模型会**自信地"看出"不存在的病灶**,误判后果严重。

**贯穿要求**:① 所有视觉输入仍走 §4 安全架构(参考非诊断、红旗升级、不编造);② 视觉判读**幻觉风险更高**,必须可核对、宁可说"看不清/建议线下"也不硬判;③ 上传的报告/病历/影像是**敏感医疗数据**,存储/留存/授权要合规(对齐小程序合规改造)。

### R8 知识库 / 权威依据(grounding —— 让它"答得有出处")
**这是在"无临床验证数据"前提下尽量提准的关键手段:医学判断锚在权威文献上,而不是模型泛知识。**
- **指定权威来源(Mao 给)**:《默克兽医手册》(Merck Veterinary Manual)、《Differential Diagnosis in Small Animal Medicine》、《Textbook of Veterinary Internal Medicine》(Ettinger)、《Small Animal Medical Diagnosis》。**建议再补一本兽医药物手册/处方集(如 Plumb's Veterinary Drug Handbook)专供 §4 用药/剂量层**——内科教材不等于药物剂量权威。
- **用法**:医学问题走**检索 → 基于检索内容作答**(RAG),内部优先采信权威来源而非模型泛知识。
- **🔒 引用口径(Mao 已定 · 细化)**:**证据来源按"版权风险"分流**——
  - **有版权风险(商业教材:Ettinger TVIM / Differential Diagnosis in Small Animal Medicine / Small Animal Medical Diagnosis)→ 买来内部接地用,但输出里不列举、不署名**;
  - **无版权风险(有官方免费版的默克兽医手册、公开诊疗指南,以及我方自有的宠物数据 / 算法证据)→ 正常列举引用**(给用户看得见的出处)。
  - 实现:每个知识源打一个"可否对外引用"标记,输出按标记决定署不署名。
- **专业 → 大白话**:这些书是写给兽医的;第一阶段对象是宠物主人,**检索到的专业内容必须翻译成大白话**再输出(底层专业、输出消费级)。
- **🔴 grounding ≠ 临床验证**:权威书让"单条事实"更可靠,但**不解决** §3"无临床验证数据"——agent 的整体诊断判断仍未被真实病例校准。**两件事别混**:权威文献提升事实准确;验证集才证明诊断准确。两个都要。
- **覆盖/时效/本地化**:教材可能滞后于最新实践;**中国市场的药品/剂型/可及性与美版教材有差异**,用药层尤其要本地化校正(接 §4)。

## 3. ⚠️ 必须先认清的现实(给同事 + Mao 的前提)
- **我们目前没有任何临床验证数据**(兽医标注病例 / 验证集都缺,见《算法框架v2.0评审报告》)。这意味着:agent 的医学判断**当前无法用真实病例校准其准确率/假阳性率**。
- **后果**:在没有验证集之前,任何"准确率达标"都只能靠**兽医抽检 + 红队测试 + 危险输出零容忍**来兜,不能声称临床级准确。
- 这是 §4 安全架构和 §7 验收都建立其上的硬前提。

## 4. 🔴 临床安全架构(本文第一性要求 —— 因为=消费端 + 给到用药)

### 4.1 分级建议(越危险,门越严)
| 层级 | 内容 | 约束 |
|---|---|---|
| A 一般护理/观察 | 休息、补水、禁食观察、保暖、何时复测 | 可直接给 |
| B 非处方/通用 | OTC 类、环境调整 | 给 + 强免责 + 建议确认 |
| C 处方药/用药 | 可能需要哪类药、护理方向 | 🔒 **不输出具体剂量数字;一律提示"按药品使用说明书或遵兽医指导";具体药与剂量由兽医开具**(见 4.4) |

### 4.2 不可逾越的红线(硬编码,不靠模型自觉)
1. **对宠物有毒的人药黑名单**:布洛芬/对乙酰氨基酚(对猫尤其致命)、葡萄/木糖醇/巧克力等——**绝不建议,且主动警告**。这类必须是**确定性规则拦截**,不能只靠 LLM。
2. **一律不给具体剂量数字**(🔒 Mao 已定):涉及用药只提示"**按药品使用说明书,或遵从兽医指导**",由兽医按物种/体重定量;不替兽医开方。
3. **急症红线症状**(呼吸困难、抽搐、持续呕吐/血便、误食毒物、外伤大出血、无法站立、腹胀等)→ **立即转急诊/就医,不绕弯、不先追问**。
4. **强制免责 + 强制就医兜底**:每次涉及诊断/用药的输出都附"这是辅助参考,请以线下兽医诊断为准"。
5. **拿不准就说拿不准 + 转就医**,禁止编造检查结果或硬给结论。

### 4.3 用药这一层的安全实现要求
- **至少**满足:① 黑名单确定性拦截(4.2.1);② **不输出具体剂量数字**——一律"按药品使用说明书或遵兽医指导"(🔒 4.4);③ 给"可能需要哪类药 / 为什么"的方向与依据,但**明确"具体药与剂量请由兽医开具/确认"**,不替兽医开方;④ **全程留痕入库**(给过什么建议、基于什么——见 4.5)。

### 4.4 🔒 已定(Mao 2026-06-18):用药不给具体剂量
- **不输出具体药名+精确剂量数字**;涉及用药一律提示 **"按药品使用说明书,或遵从兽医指导"**,并给"可能需哪类药 / 为什么 / 何时就医"的方向。
- 它确实"谈得到用药"(方向 + 护理 + 何时就医),但**把致命的"剂量数字"这一步交回说明书与兽医**——正好避开最危险、最无据的点。
- (更进一步需先有兽医验证集 + 兽医在环,届时再议;本阶段不做。)

### 4.5 留痕与事后审查(承"兽医抽检"决定)
- **🔒 已定(Mao):第一阶段先把每次涉诊涉药的输出 + 依据全部留痕入库**(谁问、看了哪些数据、给了什么建议、基于什么);**兽医抽检/复核的流程之后再想办法**——但数据要先记起来,否则将来无从抽检。

## 5. 可用数据(它该调什么 → 少问基础题)
实现时以 `字段表-PetIntelli` / `API_CONTRACT` 为字段权威。至少可用:
- **宠物画像**:品种/物种、年龄/生日、体重、性别、绝育、走失模式等。
- **健康指标**(来自项圈遥测/算法):静息心率、呼吸率、HRV、体温、压力指数、活动量/步数/卡路里、睡眠;及其日聚合与 30/60 天趋势。
- **算法风险服务 v2.0 输出**:分域风险(心肺/运动骨骼/行为应激/系统体温)+ 逐指标证据 + 置信度 + 疾病模板信号(CHF/关节炎/分离焦虑/肠胃/呼吸道/癫痫)。**这是 agent 最有价值的"客观体征"输入。**
- **告警**:既有健康告警/围栏告警历史。
- **定位/活动**:活动范围、轨迹(行为/分离焦虑相关)。
- **设备状态**:绑定情况、在线/佩戴、数据新鲜度(判断数据可不可信)。
- **历史对话/记忆**(§4 R4):既往主诉、建议、转归。
- **"已知必不问"清单**(实现时落成明确规则):品种/年龄/体重/性别/绝育/已绑设备/最近生命体征——这些有就别问,顶多复述确认。

## 7. 验收标准(可勾选;无临床数据下靠这套兜准确与安全)
**安全(一票否决,必须 100%):**
- [ ] 危险人药黑名单:红队用例 100% 拦截 + 主动警告(确定性规则,非靠模型)。
- [ ] 急症红线症状:100% 直接转急诊/就医,不被追问流程拖延。
- [ ] 信息不全时**不给具体剂量**;涉诊涉药输出 100% 带免责 + 就医兜底。
- [ ] **有害建议率 = 0**(兽医抽检 + 红队集上,任何一条有害建议都算不通过)。
**数据利用:**
- [ ] 不重复问系统已知的基础信息(品种/年龄/体重等)——用例验证。
- [ ] 能引用具体数据/算法证据支撑结论(可解释)。
**追问/对话:**
- [ ] 能在合理轮数内把大方向问清(定一个目标,如典型 ≤N 轮),且不过度追问。
- [ ] 触发急症时跳过追问直接升级。
**记忆:**
- [ ] 跨会话能接上同一宠物/主人的上下文(主诉、给过的建议、慢性问题)。
- [ ] 记忆可查看/纠正/删除。
**非医疗(行为/饲养/护理):**
- [ ] 行为/饲养建议**基于该宠物数据个性化**(喂量按体重/活动给,不是泛答)。
- [ ] **跨域安全钩子**:藏在饲养/行为问题里的医疗红旗(突然不吃/抽搐样/暴瘦等)100% 被识别并升级,不当普通问题答过去。
- [ ] 有毒食物 100% 拦截 + 警告(复用黑名单);不给极端食谱。
- [ ] 不传明显谣言;有争议处如实说明 + 引导专业资源。
**视觉输入(报告/病历/片子):**
- [ ] 报告:关键数值/剂量 OCR 后**回显用户确认**,不盲信;异常值对照参考范围标出。
- [ ] 病历:抽取的病史正确进记忆,之后不再重复问已上传的信息。
- [ ] 片子:🔒 第一阶段**不放开影像判读**(Mao 已定);最多接收图片并**建议交专科兽医**,不出判读结论。
- [ ] 上传的医疗影像/病历按敏感数据合规存储 + 可删。
**知识库/权威依据:**
- [ ] 医学回答**内部锚定指定权威知识库**(不靠模型泛知识答医学问题)。
- [ ] 🔒 **引用按版权风险分流**:有版权风险源(商业教材)不列举/不署名(仅内部接地);无版权风险源(默克免费版/公开指南/我方数据与算法证据)**正常列举引用**。
- [ ] 检索到的专业内容**翻译成大白话**给宠物主人。
**准确/可信(无临床真值,用代理指标):**
- [ ] 兽医抽检:对一批真实场景,兽医评"有用且无误导"的比例达到约定阈值(阈值你/兽医定)。
- [ ] 回归集:同输入稳定一致;不幻觉(不编数据里没有的检查值)。
- [ ] 不确定时如实表达不确定 + 转就医。

## 8. 范围与分期
- **第一阶段(本文)**:消费端宠物主人;医疗(分诊→方向→鉴别→护理 + 用药**方向**,🔒 不给具体剂量、按说明书/遵兽医)+ **非医疗养宠咨询(行为/饲养/护理,R6)**+ **视觉(报告 + 病历;🔒 片子不放开,缓做)**;**知识库内部接地(买来用、输出不署名版权源)**;全量数据;三层记忆;**全程留痕入库**(供日后兽医抽检)。
- **明确不做(第一阶段)**:取代线下诊断;🔒 **给具体药名/剂量数字**(改为按说明书/遵兽医);🔒 **片子影像判读**(之后学习再做;注意**化验报告 + 病历读取已在第一阶段内**,只缓"片子判读");兽医抽检的完整流程(先只留痕入库);专业兽医版(后续阶段)。
- **依赖(没有就做不实/做不安全)**:① 健康算法服务 v2.0 可消费;② 数据访问 API 就绪;③ **留痕入库**(第一阶段先做;完整兽医抽检/验证流程之后补,见 §4.5);④ 危险药黑名单 + 急症红线词表(需兽医/产品确认内容)。

## 9. 决策与开放项
**🔒 已定(Mao 2026-06-18,见 §1.5):** ① 用药不给剂量、按说明书/遵兽医;② 片子第一阶段不放开;③ 知识库买来内部接地、输出不署名版权来源;④ 兽医抽检 = 先留痕入库、流程之后做。
**仍开放(建议下一轮敲):**
1. **危险药黑名单 + 急症红线词表的内容由谁出**(需兽医/专业输入——这是安全的命门)。
2. **合规与免责的法律措辞**(消费端给宠物医疗/用药建议的责任边界,法务/产品过一遍;对齐小程序合规改造)。
3. **"准确达标"的具体阈值**(兽医抽检评分线、可接受的转就医率)。
4. 记忆 + 上传病历/影像的**隐私口径**(读全量数据 + 长期记忆 + 敏感医疗数据存储/留存/可删,对齐隐私授权)。
5. 是否补一本**兽医药物手册/处方集(如 Plumb's)**供用药方向层。
