## 1. 宠物画像（只读，对话开始时拉取）

> 拉取须带 **`pet_id`** 查询参数（PRD FR-DATA-05、§5.2.7）。

| 字段 | 语义 | 第一阶段必需 | 缺失时行为（见 PRD §8.4） |
| --- | --- | --- | --- |
| `species` | 物种（犬 / 猫 / 其他） | **是** | 检验 P2 降级 P4；须追问确认 |
| `breed` | 品种 | 否 | 用物种级默认 |
| `birth_date` 或 `age` | 生日或年龄 | 强烈建议 | 生命阶段未知 → 参考区间 P2 降级 P4 |
| `weight_kg` | 体重（kg） | 强烈建议 | FR-ASK-04 追问确认；不禁用 `standard` |
| `sex` | 性别 | 否 | 可追问 |
| `neutered` | 是否绝育 | 否 | 可追问 |

## 2. 主人级记忆（须 CRUD）

| 字段 / 能力 | 语义 | 第一阶段必需 |
| --- | --- | --- |
| 沟通偏好 | 详略、语气等 | 否（有则使用） |
| `pets[]` | 宠物列表：`{ pet_id, display_name, species, … }` | **是**（多宠家庭） |
| `default_pet_id` | 无指代时的默认咨询对象（PRD §5.2.7.3 P6-D1） | **是**（多宠时强烈建议） |
| 关注点 / 风险偏好 | 主人关注主题 | 否 |
| **写入能力** | 跨会话持久化、用户可查看 / 纠正 / 删除 | **是** |

> **当前咨询对象** 为每轮 **`resolved_pet_id`**（由 Agent 定宠，PRD §5.2.7），非静态「当前对话关联宠物」单字段。
