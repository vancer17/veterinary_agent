使用 **前置检索、阶段式按需检索、结果后校验检索**三层调用模式，结合四层递进问诊+Slot/FSM/KG架构落地。

## 二、RAG 调用时机与各阶段实现细节
### 1. 前置初始化检索（用户初次主诉时执行）
#### 目标
基于用户初始主诉、基础人口学Slot（年龄、性别、既往史）做第一轮多路召回，锁定当前最相关的诊疗知识库范围，限定后续所有问诊、分诊的知识边界。
1. **Query构造方式**
- 结构化Query：用户主诉 + 年龄、性别、基础病史 + 就诊诉求；
- 非结构化Query：纠错后的用户原始症状描述；
- 多路召回：向量相似度检索 + BM25关键词检索 + 元数据过滤（科室、疾病大类、风险等级）。
2. **过滤策略**
- 相似度阈值过滤（通常0.6），丢弃低相关文档片段；
- 按文档权威度权重重排：临床指南 > 院内诊疗规范 > 科普类资料；
- 聚合去重，保留Top5~Top10条证据片段存入会话上下文的`global_rag_evidence`只读缓存，本轮会话全局复用。
3. **业务作用**
限定问诊方向，避免跨科室无关提问；同时为后续紧急度初判提供权威依据。

### 2. 问诊候选问题生成阶段：RAG 赋能高价值追问筛选
#### 实现方式
1. 基于当前已收集的症状Slot，结合全局缓存的RAG诊疗指南片段，提取该类疾病**关键鉴别体征、高危伴随症状、必采既往禁忌信息**；
2. 给每个Slot配置关联的RAG标签，优先召回指南中高频必填采集项，拉高对应Slot优先级；
3. 约束：生成的1~3个追问必须在召回的诊疗证据覆盖范围内，禁止脱离权威知识库随意提问。
4. 溯源埋点：每条追问绑定对应的RAG文档ID、片段位置，记录「为什么要问该问题」的临床依据，用于安全审查逻辑链补丁。

### 3. 第二层（紧急度+分诊）触发阶段：定向RAG检索
当`slot_coverage`达标/FSM流转到分诊状态时，执行**定向精准检索**：
1. 检索Query：当前已结构化的全部症状集合 + 急症判定关键词；
2. 检索范围：限定「急诊预检分诊规范、急症识别指南」类文档；
3. 从召回片段中匹配急症判定规则（胸痛、呼吸困难、意识障碍、大出血等高危指征）：
   - 命中高危证据 → 紧急度标记危急/高风险，直接熔断第四层居家用药建议；
   - 匹配科室诊疗范围 → 输出推荐分诊科室；
4. 强制规则：**紧急度等级、分诊科室必须至少绑定一条RAG原文证据**，无权威依据则不输出分诊结论，继续补充问诊。

### 4. 第三层（疾病大类+鉴别诊断）：多证据融合检索
信息充足解锁本层时，做聚焦式RAG召回：
1. Query：已采集全部症状+分诊科室；
2. 召回该科室下常见疾病临床描述、疾病鉴别要点文档；
3. 候选疾病必须在召回文档内出现，每条可能疾病方向、鉴别诊断均附上对应原文片段作为依据；
4. 规避幻觉：模型只允许从召回的Top检索结果内做归纳总结，不允许凭空编造疾病、鉴别要点；
5. 多条相似片段做融合摘要，精简后作为诊断依据写入结构化审计草稿。

### 5. 第四层（居家/非处方建议）：强约束定向检索
本层是风控最高的RAG调用场景，必须双重检索校验：
1. 第一路检索：用药禁忌、过敏风险、特定人群（孕妇、老人、肝肾功能不全）OTC用药规范；
2. 第二路检索：对应疾病居家护理、物理处置、饮食休息类临床建议文档；
3. 两条强约束：
   - 必须同时召回「无用药禁忌」相关证据，才可推荐非处方药；
   - 所有护理、用药建议逐句匹配RAG原文，禁止模型改写生成超出指南范围的方案；
4. 只要任意一条禁忌类文档命中当前用户基础情况，直接禁止输出用药建议，仅保留通用护理宣教。

### 6. 后置风控校验检索（全场景兜底）
每轮Agent输出所有内容后，执行一次安全校验类RAG检索：
1. 检索医疗红线知识库：禁止诊疗话术、处方药推荐、确诊类违规表述；
2. 若召回匹配到风险红线规则，直接拦截输出，替换为标准化就医提示；
3. 所有输出内容、拦截原因、命中的风险文档全部写入逻辑链审计补丁。

## 三、会话级RAG缓存与复用设计（避免重复检索、降低耗时）
### 1. 三级RAG存储隔离
1. **全局会话RAG缓存（只读）**
用户初次主诉检索后的Top权威文档片段，存储在Redis会话维度，全程复用，避免每轮追问都全量检索，优化接口耗时；
2. **阶段局部RAG缓存**
分诊、鉴别、护理每个阶段单独存储本轮定向检索的证据，和全局缓存做差异补充；
3. **单结论绑定证据映射表**
每一条输出内容（追问、紧急度、疾病、建议）通过ID绑定对应的一条或多条RAG片段，用于审计溯源。

### 2. 增量更新策略
- 用户每轮补充关键症状后，触发一次增量RAG检索，补充新的相关文档，合并去重更新全局缓存；
- 若仅补充次要信息、无核心症状变更，直接复用已有缓存，不再重复检索。

## 四、多路召回+重排的工程标准配置（医疗场景专用）
1. **召回策略组合**
- 稠密向量召回（BGE医疗微调模型）：语义匹配用户症状描述；
- 稀疏BM25召回：精准匹配疾病、症状、科室专业关键词；
- 元数据过滤召回：按科室、风险等级、文档类型做前置过滤，缩小检索域。
2. **重排Rerank**
使用医疗领域微调重排模型对多路召回结果做打分排序，剔除语义相似但权威性低的片段，保证靠前片段均为临床规范内容。
3. **上下文窗口压缩**
将Top权威片段做摘要压缩后注入Prompt，防止超长上下文溢出，同时保留核心诊疗规则。

## 五、结合Slot/FSM/KG三种架构的差异化RAG调用方式
### 1. Slot架构场景
1. 每一轮收集完Slot后，用已填充的结构化Slot拼接检索Query；
2. 用RAG返回的临床维度校验当前缺失Slot的优先级，决定1~3个追问产出；
3. 每一层解锁均依赖对应领域的RAG检索结果做准入判定。

### 2. FSM有限状态机架构
1. 每个状态节点绑定专属的检索知识库索引：
   - 问诊状态：通用症状采集指南库；
   - 分诊状态：急诊预检规范库；
   - 鉴别诊断：专科疾病指南库；
   - 护理建议：OTC与居家护理库；
2. 流转进入某一状态时，自动在对应知识库内做定向检索，避免跨库噪声。

### 3. 知识图谱+RAG混合架构
1. 知识图谱完成症状-疾病推理、候选问题筛选；
2. 图谱推理得到的实体、疾病作为关键词构造RAG Query，反向召回权威临床原文；
3. 图谱负责决策逻辑，RAG负责结论证据接地与话术规范化，每条推理路径绑定检索片段。

## 六、RAG证据结构化留存（满足安全审查留痕要求）
### 1. 结构化草稿中RAG标准存储结构
```json
"rag_evidence": [
    {
        "evidence_id": "rag_001",
        "source_doc": "成人急性头痛预检分诊临床指南V2.0",
        "doc_type": "临床指南",
        "similar_score": 0.89,
        "quote_segment": "剧烈头痛伴随呕吐需判定为中高风险，优先神经内科就诊",
        "bind_content": "本次紧急度判定为中风险，推荐神经内科分诊"
    }
]
```
### 2. 逻辑链补丁记录要点
1. 本轮RAG检索的Query、召回策略、过滤阈值、重排后命中条目；
2. 哪些结论绑定了哪条证据，未输出某类建议的RAG依据（如命中用药禁忌文档）；
3. 复用历史缓存证据还是增量新检索，检索耗时、命中数量。

## 七、工程避坑核心规范
1. **禁止无依据自由生成**：任何医疗类输出必须绑定至少一条RAG权威片段，无检索结果时只能继续追问，不能给出推测性结论；
2. **检索域隔离**：高危急症、用药、普通科普三类知识库物理索引隔离，定向检索防止越界匹配；
3. **只读约束**：所有RAG检索均为只读操作，仅查询向量库与文档库，不会修改知识库原始数据；
4. **敏感内容过滤**：召回结果前置脱敏，剔除隐私、处方药剂量等违规片段再注入大模型Prompt；
5. **超时熔断降级**：RAG检索超时后，直接复用历史会话缓存证据，不再生成高阶诊断与用药建议，仅保留问诊追问与风险提示。
